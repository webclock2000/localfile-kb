## 阅读路径

### 评审专家

先读 PRD 了解产品要解决的问题和价值主张，再读 DEVELOPMENT_V3.md 的 §1-4（概览、目标、架构决策、系统架构）理解技术方案。

### 开发人员

以 DEVELOPMENT_V3.md 为唯一开发依据，按目录顺序完整阅读。PRD.md 提供需求背景参考。

## 文档格式

- **Word (.docx)** — 适合评审专家正式审阅，格式排版规范
- **Markdown (.md)** — 适合开发人员日常参考，支持版本管理和 diff

两套格式内容完全一致。
